# Blockly Bot 仕様

Blockly Bot は **JS コードを生成して Bot Runtime で実行**します。
このドキュメントは「ブロックの意味」と「実行ルール」を実装準拠でまとめます。

## 実行モデル

- 1ターンに **1アクション**のみ
- `api.around` の参照は行動を消費しない
- `look` / `search` も **行動**として 1ターンを消費する
- 行動後は **必ず `ターンを終える` ブロック**で終了する
  - 未配置の場合は `chaser_on_turn` のガードで実行時エラーになる

## 主要ブロック

### ターン開始ブロック
- **毎ターン** (`chaser_on_turn`)
  - ここからロジックが開始される
  - 1つだけ配置できる（UI 側で強制）

- **最初に1回だけ** (`chaser_on_start`)
  - `onTurn` の外側で一度だけ実行される
  - `api` が存在しないため **行動ブロックは置けない**（置くと実行時エラー）
  - 初期化（変数の初期値設定など）に使用

### 行動ブロック
- **歩く / ブロックを置く**
  - 文ブロック（statement）
  - そのターンの行動として確定

- **見る / まっすぐ見る**
  - 式ブロック（reporter）
  - 結果配列を返すが、**それ自体が行動**

- **見る結果を変数に入れる / 探索結果を変数に入れる**
  - 観測結果を変数に保存する専用ブロック
  - そのターンの行動として確定

- **ターンを終える**
  - 1ターン1アクションの終点

### 状態・補助ブロック
- **変数**
  - 生成される JS ではグローバル変数
  - 初期値は `0`
  - ターンを跨いで保持される

- **現在のターン数**
  - `__chaserTurn` を参照（`chaser_on_turn` で加算）

- **前回の方向**
  - `__chaserLastAction` を参照

- **ランダムに歩く**
  - `api.around` を使ってブロックを避ける
  - 4方向がブロックならランダムに歩く

## 実装参照

- ブロック定義 / 生成ルール: `src/lib/blockly/chaserBlocks.ts`
- Blockly エディタ: `src/app/my/bots/blockly/[id]/BlocklyBotEditorClient.tsx`
