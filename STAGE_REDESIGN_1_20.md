# STAGE_REDESIGN_1_20.md — ステージ01〜20 再設計

作成日: 2026-05-13（v2）

## 前提：反応型プログラミングの難易度構造

| 難易度 | 内容 |
|---|---|
| 低 | 毎ターン同じ行動（条件なし） |
| 中 | 隣のマスを見てif/else（1条件） |
| 高 | 複数条件・同じルールを2〜4バリアントに適用 |
| より高 | 変数で過去の判断を保持 |

**迷路が複雑でもコードが簡単になる例:**
→ 長い蛇行通路でも、「上に壁があれば右、なければ上」の1条件で全部解ける

**見た目が単純でもコードが難しい例:**
→ 同じ見た目の分岐が2か所ある迷路（変数なしでは区別不可）

## 新ブロックセット定義

| セット名 | 追加されるブロック | 解禁 |
|---|---|---|
| `BLOCKS_BASIC` | `chaser_on_turn`, `chaser_action_walk`, `chaser_turn_end` | 01〜 |
| `BLOCKS_CHECK` | + `controls_if`, `logic_compare`, `chaser_is_tile`, `chaser_tile_value`, `chaser_get_tile`, `math_number` | 04〜 |
| `BLOCKS_STATE` | + `chaser_state_*`, `chaser_last_direction`, `chaser_direction_value`, `chaser_get_around`, `chaser_action_walk_last`, `chaser_turn_number`, `math_modulo`, `logic_operation` | 10〜 |

※ `controls_if` と `chaser_is_tile` を同時解禁する（既存設計の修正点）
